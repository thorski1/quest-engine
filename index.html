Architecture

Clean separation of engine and content

Quickstart

Build your first pack in 5 minutes

Three files. One command. A fully functional terminal learning game.

1
Install the engine
# From PyPI pip install quest-engine # Or from source pip install -e ./quest-engine
2
Write your zones and challenges
skill-packs/my_pack/zones.py python
from engine.zone import Zone from engine.challenge import Challenge ZONE_ORDER = ["intro", "advanced"] ZONES = { "intro": Zone( id="intro", title="Getting Started", description="The basics of our topic.", challenges=[ Challenge( id="q1", type="quiz", prompt="Which command lists directory contents?", options=["ls", "cd", "pwd", "cat"], answer="a", explanation="`ls` lists files in the current directory.", hints=["Think 'list'.", "Two letters."], xp=10, ), Challenge( id="q2", type="fill_blank", prompt="The command ___ prints the working directory.", answer="pwd", explanation="`pwd` = Print Working Directory.", xp=10, ), ], ), }
3
Define your SkillPack
skill-packs/my_pack/__init__.py python
from engine.skill_pack import SkillPack from .story import INTRO_STORY, ZONE_INTROS, ZONE_COMPLETIONS, BOSS_INTROS from .zones import ZONES, ZONE_ORDER SKILL_PACK = SkillPack( id="my_pack", title="My Learning Pack", subtitle="◈ A subtitle ◈", save_file_name="my_pack", intro_story=INTRO_STORY, quit_message="Progress saved. See you soon!", zone_order=ZONE_ORDER, zones=ZONES, zone_intros=ZONE_INTROS, zone_completions=ZONE_COMPLETIONS, boss_intros=BOSS_INTROS, zone_achievement_map={"intro": "first_steps"}, achievements={"first_steps": ("First Steps", "Completed the intro!")}, level_titles=[(1, "Novice"), (5, "Learner"), (10, "Expert")], kids_mode=False, )
4
Wire up your entry point
your_game/main.py python
import os from pathlib import Path from engine.main import run _HERE = Path(__file__).parent.parent os.environ.setdefault("QUEST_SKILL_PACKS_DIR", str(_HERE / "skill-packs")) def main(): run("my_pack")
5
Run it
pip install -e . my-game
Challenge Types

Six ways to test knowledge

Mix challenge types within a zone to keep players engaged.

quiz
Multiple Choice
Player types A / B / C / D. The most common type — fast to answer, easy to write.
fill_blank
Fill in the Blank
Player types the exact answer. Case-insensitive. Good for commands, terms, and numbers.
flag_quiz
Multiple Valid Answers
Like fill_blank but accepts any answer from an answers list. Perfect for CLI flags where -a and --all are both right.
live
Real Shell Command
Player runs an actual command in a sandboxed temp directory. The engine validates the result: file created, output contains, permissions set, etc.
ordered
Sequence Ordering
Player types the correct order of steps (e.g. "3 1 0 2"). Great for workflows like git add → commit → push.
arrange
Matching Pairs
Player matches left items to right items. Good for pairing commands to descriptions or terms to definitions.
Built-In Features

Everything the game loop needs

Zero configuration. Everything is included and works out of the box.

Progression
XP & Levels
Configurable level titles per pack. 300 XP per level. Difficulty multipliers: Easy 0.75×, Normal 1×, Hard 1.5×.
Habit
Daily Challenge + Streaks
Deterministic daily pick per pack. 2× XP bonus. Streak tracking with max streak. Persists across sessions.
Quality
Zone Star Ratings
1–3 stars per zone based on hints used and challenges skipped. Players can replay to earn perfect scores.
Mastery
Weak Spot Review
Tracks every wrong answer. Review mode drills the player on their personal weak spots.
Competition
Speed Records
Personal best time tracked per challenge. New records flash on screen with a bonus XP indicator.
Retention
Bookmarks
Players toggle [b] on any challenge. A review menu replays all bookmarked challenges.
Narrative
Campaigns
Chain multiple packs into a story with chapter intros, outros, and an optional placement quiz to pick starting chapter.
Completion
Grade Certificate
ASCII certificate with S/A/B/C/D grade on pack or campaign completion, based on accuracy and star performance.
Distribution
Auto-Updates
Checks PyPI at launch with a 24-hour cache. One keypress installs and restarts with the new version.
Audience
Kids Mode
Set kids_mode=True on any pack for warm, encouraging praise phrases instead of the default responses.
Analytics
PostHog Integration
Optional PostHog event tracking. Anonymous player IDs. Disable with QUEST_NO_ANALYTICS=1.
Save System
JSON Persistence
All progress saved to ~/.quest_engine/<pack_id>/. Resume any session exactly where you left off.
In-Game Controls

Standard keybindings

Every pack gets these controls automatically — no configuration needed.
